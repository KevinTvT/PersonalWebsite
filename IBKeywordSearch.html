IB Keyword Search

This is a project that was created to help teachers in the IB program efficiently and easily create tests from past papers.

This is an image of the IB Keyword Search main Graphical User Interface.

Instructions Tab

Keeping in mind that the program may not be intuitive to every single teacher, I created an Instructions Button that would open an Instructions Tab with directions and common errors. These Instructions also has information relating to the change folder accessed function of the program as this part of the program can be especially confusing.

Change Folder Accessed Tab

Keeping in mind that some users may rather store the papers in a location rather than the default location, I created this tab that writes to a file and changes the location the program is accessing. Each name in the list is a link, in order to change the folder accessed to that folder the user just needs to click the link and the "Done" button when finished.

Main Graphical User Interface

In order to display that the program is working when the user searches something, the program displays a loading alert. This loading alert is run using multithreading, while searching through the tests, the program simultaneously calculates the progess of the search.

Output

After the program is done searching, it outputs the tests separated into 6 groups paper 1s, 2s, 3s, and each of their markschemes. Each paper name in the list is clickable and will open the exam file as a pdf in the user's default pdf viewer. The questions where the keyword was found will update in the question number tab when an exam is opened.

Error Handling

The program may run into errors such as "Keyword was not found!" When the program runs into errors like these it will catch and an alert tab will pop up with the respective error message allowing the user to adjust the search term in this case. This is just one example of the error handling the program does.
